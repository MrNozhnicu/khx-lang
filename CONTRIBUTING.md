# Вклад в KHX

Спасибо за интерес к развитию KHX! Мы рады любому вкладу.

## Как внести вклад

### 1. Сообщить об ошибке

Если вы нашли ошибку:
- Проверьте, не была ли она уже сообщена в [Issues](https://github.com/yourusername/khx-lang/issues)
- Создайте новый Issue с подробным описанием
- Укажите версию KHX и Python
- Приложите код для воспроизведения

### 2. Предложить улучшение

Есть идея? Создайте Issue с тегом `enhancement`:
- Опишите предлагаемую функциональность
- Объясните, зачем это нужно
- Приведите примеры использования

### 3. Написать код

1. Fork репозитория
2. Создайте ветку для вашей функции:
   ```bash
   git checkout -b feature/amazing-feature
   ```
3. Внесите изменения
4. Добавьте тесты (если применимо)
5. Commit изменения:
   ```bash
   git commit -m "Add amazing feature"
   ```
6. Push в вашу ветку:
   ```bash
   git push origin feature/amazing-feature
   ```
7. Откройте Pull Request

## Стиль кода

- Следуйте PEP 8 для Python кода
- Используйте понятные имена переменных
- Добавляйте комментарии для сложной логики
- Документируйте новые функции

## Структура модулей

Каждый модуль KHX должен:
- Иметь docstring с описанием
- Экспортировать функции через `__all__`
- Включать тесты в `if __name__ == "__main__"`
- Следовать паттерну хранения объектов

Пример:
```python
"""
KHX Module Name
Описание модуля
"""

# Storage
objects = {}
counter = 0

class MyClass:
    def __init__(self):
        pass

def create_object():
    global counter
    obj_id = f"obj_{counter}"
    counter += 1
    objects[obj_id] = MyClass()
    return obj_id

def get_object(obj_id):
    return objects.get(obj_id)

if __name__ == "__main__":
    # Тесты
    pass
```

## Тестирование

Перед отправкой PR:
1. Запустите все примеры из `examples/`
2. Проверьте, что редактор работает
3. Убедитесь, что новый код не ломает существующий

## Документация

При добавлении новых функций:
- Обновите README.md
- Добавьте примеры в `examples/`
- Документируйте API
